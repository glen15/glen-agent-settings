# TDD

tdd-guide 에이전트를 호출하여 Red → Green → Refactor 사이클을 적용한다.

## 사이클

```
RED → GREEN → REFACTOR → REPEAT
```

1. 실패하는 테스트 작성 (구현 없이)
2. 테스트 실행 — 실패 확인
3. 통과할 최소한의 코드 작성
4. 테스트 실행 — 통과 확인
5. 리팩토링 (테스트 green 유지)
6. 커버리지 확인 (80%+)

## 커버리지 필수 대상

- 금융 계산, 인증, 보안: **100%**
- 나머지: **80%+**

- `/code-review` — 구현 완료 후
- `/test-coverage` — 커버리지 부족 시
