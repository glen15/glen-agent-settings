# Plan

코드 작성 전에 구현 계획을 세운다. 내장 Plan 모드를 활용한다.

## 프로세스

1. 요구사항을 명확한 용어로 재확인
2. 기존 코드베이스 분석 (영향받는 파일, 패턴)
3. 단계별 분해 (파일 경로, 액션, 의존성, 위험도)
4. 사용자에게 계획 제시 → **명시적 승인 전까지 코드 작성 금지**

## 다음 단계

- 승인 후 `/tdd`로 테스트 주도 구현
- 완료 후 `/code-review`로 검토
